# Search V2 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make the country sidebar search easier to use with deterministic aliases, suggestions under the input, and debounced filtering.

**Architecture:** Add one focused search utility in `src/lib/data/searchSuggestions.ts` that owns normalization, alias expansion, match text, and suggestion ranking. Keep `FaroExperience` responsible for debounce and state, and `CountrySidebar` responsible only for rendering the input and suggestion actions.

**Tech Stack:** Next.js/React client components, Node test runner, existing Faro data/signal types. No new dependencies.
