# Deployment

Faro should use Vercel as the public app host for now.

The current app is a Next.js project with repo-versioned generated data. It does not require a runtime database, Prisma, Drizzle, or migrations to serve the product. Vercel gives the project automatic deploys from `main`, preview deploys for branches, and enough build/runtime capacity for the app as long as API responses stay bounded.

## Production Shape

- Public app: Vercel, currently `faro-ilumina.vercel.app`.
- Data artifacts: generated during `npm run build`.
- Heavy exports: static files under `public/exports/`, generated by `npm run exports:build` before `next build`.
- Hetzner VPS: reserved for ETL, cron jobs, object storage, Postgres/PostGIS, and heavier data services when Faro outgrows repo-versioned data.

## Vercel Function Limits

Do not return full data collections from Vercel Functions. Vercel Functions have a hard response payload limit, so collection endpoints must stay paginated or redirect to static artifacts.

Current API contract:

- `/api/cases` returns paginated case collections, default `limit=100`, max `limit=200`.
- `/api/signals` returns paginated signal rows, default `limit=100`, max `limit=200`.
- `/api/export/[id]` returns one case evidence pack and can remain a Function response.
- `/api/export` redirects core exports to generated static files when available.
- `/api/export` returns a bounded manifest for ad hoc large filtered exports that do not have a static artifact.

## Build Commands

```bash
npm run data:verify
npm test
npm run typecheck
npm run build
```

`npm run build` runs `npm run exports:build` first, so Vercel deploys include the generated static export artifacts without committing those large generated files.

## Hetzner Boundary

Use the Hetzner skill before touching the VPS. Administrative SSH should go through Tailscale only. Do not move the public UI to Hetzner unless there is a concrete reason such as long-running jobs, a database service, private object storage, or operational constraints that Vercel cannot cover cleanly.
